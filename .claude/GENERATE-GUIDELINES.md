```
[자동 감지 결과]

- 주요 언어: [감지된 언어]
- 기본 브랜치: [감지된 브랜치]

이 정보가 맞나요? 수정이 필요하면 알려주세요.
추가로 다음 정보가 필요합니다:

1. 프로젝트 설명 (한 문장)
2. 이슈 트래커 (GitHub Issues / Jira / 기타)
3. 커밋 메시지 스타일 (Conventional Commits / 자유 형식)
4. 특별히 강조할 코드 규칙
5. 프로젝트 계획서 생성 여부
```

### Step 3: 지침 파일 생성

사용자 답변을 종합하여 다음 파일들을 생성합니다:

#### 3.1 CLAUDE.md (메인 지침)

구조:
```markdown
# Claude 수칙

## 작업 규칙
1. 코드 작성 전 이슈 등록
2. 이슈 제목/본문 형식
3. 브랜치/커밋/PR 규칙 요약
4. 브랜치 전략 요약

## 빌드 설정
- 프로젝트별 빌드 명령어
- 타겟 환경 정보

## [언어] 코드 생성 필수 규칙 (CRITICAL)
[언어별 필수 규칙 - SETUP-GUIDE.md의 템플릿 참조]

## 코드 품질 원칙 (필수 준수)
1. Dead Code 제거
2. 중복 코드 금지
3. 공통 함수화
4. 적절한 객체화
```

구조:

```markdown
| feature | feature | 새 기능 |
| ... | ... | ... |

## 브랜치명 생성 규칙
형식: <type>/#<issue-number>-<short-description>

## 규칙
- 소문자만 사용
- 하이픈(-) 사용
- 이슈 번호 필수
- 50자 이내

## 실전 예시
[프로젝트에 맞는 예시]

## 검증 규칙
[자동 검증 체크리스트]
```
